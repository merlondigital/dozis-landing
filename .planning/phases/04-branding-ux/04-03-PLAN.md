**Service Worker** (`public/sw.js`) — per D-19:
- Plain JavaScript file (not TypeScript — service workers are loaded directly by the browser)
- Cache name: `dozis-v1`
- Strategy: **stale-while-revalidate** for navigation requests (HTML pages), **cache-first** for static assets (JS, CSS, fonts, images)
- On `install` event:
  - `self.skipWaiting()`
  - Pre-cache critical assets: `/app/events`, the main CSS bundle path won't be known ahead of time so skip pre-caching specific hashed filenames
- On `activate` event:
  - `self.clients.claim()`
  - Clean up old cache versions (delete caches that don't match `dozis-v1`)
- On `fetch` event:
  - For navigation requests (`request.mode === 'navigate'`): stale-while-revalidate — try network first with a 3-second timeout, fall back to cache. This ensures the QR page works offline if previously visited.
  - For static assets (`request.destination` in `['script', 'style', 'font', 'image']`): cache-first — serve from cache if available, otherwise fetch and cache
  - For API requests (`request.url.includes('/api/')`) and non-GET: always network, never cache
  - For everything else: network-first with cache fallback
- Keep it simple — no workbox, no complex routing. Under 80 lines of JS.

**Service Worker Registration** (`app/layout.tsx`):
- Add a `<script>` tag inside the body (after children) that registers the service worker:
  ```
  <script dangerouslySetInnerHTML={{ __html: `
    if ('serviceWorker' in navigator) {
      window.addEventListener('load', function() {
        navigator.serviceWorker.register('/sw.js').catch(function() {});
      });
    }
  `}} />
  ```
- This is the simplest approach — no React component needed for SW registration
- The `catch` silently handles registration failures (e.g., in dev, localhost without HTTPS — though localhost is special-cased by browsers)
