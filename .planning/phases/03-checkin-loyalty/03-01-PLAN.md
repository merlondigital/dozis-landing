1. Create `src/lib/checkin/actions.ts` with `"use server"` directive:

   **`checkInByToken(token: string, eventId: string)`** — Main check-in action:
   - Call `requireAdmin()` — return `{ error: "Nincs jogosultsagod." }` if not admin
   - Call `getCloudflareContext()` to get env, then `verifyQrToken(token, env.BETTER_AUTH_SECRET)`
   - If `!valid` → return `{ error: "Ervenytelen QR kod.", type: "invalid" }`
   - If parsed `eventId` from token does not match the `eventId` parameter → return `{ error: "Ez a QR kod masik esemenyhez tartozik.", type: "wrong_event" }`
   - Look up registration by `registrationId` from token using Drizzle: `db.select().from(registration).where(eq(registration.id, registrationId)).limit(1)`
   - If not found → return `{ error: "Nincs regisztracio.", type: "not_found" }` (per D-13)
   - If `reg.checkedInAt !== null` → return `{ error: "Mar becsekkolva.", type: "duplicate", guestName }` (per D-14)
   - If `reg.status === "cancelled"` → return `{ error: "A regisztracio torolve lett.", type: "cancelled" }`
   - In a single logical block (D1 does not support multi-statement transactions natively, so use sequential queries):
     a. Update registration: set `checkedInAt = new Date()`, `checkedInBy = session.user.id`
     b. Look up the user to get their current `attendanceCount`
     c. If the registration `isFree` is true (this is the free 5th event): update user's `attendanceCount` to 0 (per D-21, reset after free event check-in)
     d. Else: increment user's `attendanceCount` by 1 (per D-19). If new count is now 4, do nothing extra here — the NEXT registration will check this.
   - Look up user's name: join user table by reg.userId to get `firstName` + `lastName` or `name`
   - Return `{ success: true, type: "success", guestName: "FirstName LastName", isFree: reg.isFree, newAttendanceCount }`
   - `revalidatePath` for `/app/admin/events/${eventId}/guests` and `/app/admin/events/${eventId}/scan`

   **`manualCheckInByRegistrationId(registrationId: string, eventId: string)`** — Manual check-in fallback (per D-27):
   - Call `requireAdmin()`
   - Look up registration by ID, verify it belongs to the given eventId
   - Same check-in logic as above (checkedInAt check, status check, loyalty update)
   - Return same shape as checkInByToken

2. Create `src/lib/checkin/queries.ts` with `"use server"` directive:

   **`getEventGuests(eventId: string)`** — Returns guest list for an event (per D-24, D-25):
   - Call `requireAdmin()`
   - Query: select registration joined with user table, filtered by eventId, ordered by createdAt desc
   - Return array of `{ registrationId, userId, name, email, firstName, lastName, status, isFree, registeredAt, checkedInAt }`

   **`getEventStats(eventId: string)`** — Returns event analytics (per D-28, ADMN-03):
   - Call `requireAdmin()`
   - Query registration table filtered by eventId
   - Calculate: `totalRegistered` (status='registered'), `checkedIn` (checkedInAt is not null), `noShows` (registered but no checkedInAt and event is past), `checkInRate` (percentage)
   - Return `{ totalRegistered, checkedIn, noShows, checkInRate }`

   **`getUserLoyalty(userId: string)`** — Returns loyalty info for user dashboard (per LOYL-01):
   - Call `getSession()` (user-facing, not admin-only)
   - Query user table for `attendanceCount`
   - Calculate: `progress = attendanceCount % 5` (0-4 range), `nextIsFree = attendanceCount >= 4` (per D-20)
   - Return `{ attendanceCount, progress, nextIsFree, totalFreeEvents: Math.floor(attendanceCount / 5) }`

   Wait — correction: `attendanceCount` resets to 0 after the free event (per D-21), so it is always 0-4. No need for modulo. `nextIsFree = (attendanceCount >= 4)`.

   **`getEventRegistrationCounts(eventIds: string[])`** — Bulk counts for admin event list (per D-29):
   - Query: group by eventId, count total registrations and count where checkedInAt is not null
   - Return `Map<string, { registered: number, checkedIn: number }>`
