   - Export `playErrorSound()`: Create AudioContext, generate 200Hz sine wave, 200ms duration, 0.3 volume
   - Handle AudioContext resume (browsers block autoplay until user gesture — the scanning action counts as gesture context)
   - Use a shared AudioContext instance (create once, reuse)

3. Create `components/checkin/checkin-feedback.tsx` — "use client" component (per D-06, D-07, D-08):
   - Props: `{ type: "success" | "error", message: string, guestName?: string, isFree?: boolean, onDismiss: () => void }`
   - Green background (`bg-green-500/90`) for success, red (`bg-red-500/90`) for error
   - Shows guest name prominently on success, error message on failure
   - If `isFree` is true on success, show extra label: "INGYENES ESEMENY!" in amber
   - Auto-dismiss after 3 seconds via setTimeout (per D-08), calls `onDismiss`
   - Renders as a fixed banner at top of viewport with z-40, animates in with CSS transition
   - On error, also call `navigator.vibrate?.(200)` for haptic feedback (per D-10)

4. Create `components/checkin/qr-scanner.tsx` — "use client" component (per D-01 through D-05):
   - Props: `{ eventId: string, onResult: (result: CheckInResult) => void, paused: boolean }`
   - Use `useEffect` to initialize `Html5Qrcode` instance on mount:
     - Container div with id `qr-reader` (html5-qrcode needs a DOM element ID)
     - Call `html5Qrcode.start()` with config: `{ fps: 10, qrbox: { width: 250, height: 250 }, aspectRatio: 1.0 }`
     - Use `facingMode: "environment"` to prefer back camera (per D-03)
   - On successful decode (`onScanSuccess` callback):
     - If `paused` is true, ignore (prevents rapid-fire during feedback display)
     - Call `checkInByToken(decodedText, eventId)` Server Action
     - Pass result to `onResult`
   - Cleanup: stop scanner on unmount via `html5Qrcode.stop()`
   - Include a camera toggle button that stops current camera and restarts with `facingMode: "user"` (per D-03)
   - Scanner container: full width, rounded corners, overflow hidden
   - Important: html5-qrcode is a client-only library. Import it dynamically: `const { Html5Qrcode } = await import("html5-qrcode")` inside useEffect to avoid SSR issues.
   - Continuous scan mode (per D-04): the scanner keeps running between scans. The parent manages the `paused` prop to gate processing during feedback display.

5. Create `components/checkin/manual-entry.tsx` — "use client" component (per D-15, D-16, D-17):
   - Two sections:
     a. **Token input**: Text input + submit button. On submit, call `checkInByToken(token, eventId)`. Same result handling as scanner.
     b. **Guest search**: Text input for name/email search. Client-side — receives full guest list as prop, filters by name/email match. Shows matching rows with "Becsekkoltatás" button that calls `manualCheckInByRegistrationId(registrationId, eventId)`.
   - Props: `{ eventId: string, guests: GuestRow[], onResult: (result: CheckInResult) => void }`
   - Search is case-insensitive substring match on name, firstName, lastName, email fields (per D-26)
